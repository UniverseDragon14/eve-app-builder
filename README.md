## Purpose

EVE App Builder is the app and web creator direction inside the Universal Dragon ecosystem.

The goal is simple: when Aslam asks for an app, EVE should not immediately create random code. EVE should ask short smart questions, create a small plan, wait for approval, then build step by step with backup, test, and rollback thinking.

## Project Role

- **NOVA Guardian Core:** safe brain, approval, backup, validation, rollback, learning.
- **EVE App Builder:** app and web creator workflow.
- **Universal Dragon Studio:** public creator tools and content workflow layer.
- **UDOS:** assistant-based control layer above apps, tools, files, and devices.

## Builder Workflow

```text
User idea

   ↓
```

## Approval Rule

EVE should follow this rule:

```text
Do not build immediately.
Discuss → Plan → Ask OK → Build → Test → Save.
```

## Safety Rules

- No API keys in public repos.
- No private Pi paths in public UI unless intentionally documented as generic examples.
- No unrestricted terminal in public apps.
- No destructive actions without approval.
- No hidden automation.
- No real-world hardware action without extra approval.

## File Standards

Real app projects should include:

- clear README
- simple folder structure
- error handler
- backup/rollback plan
- safe environment variable usage
- test or health check where possible
- build log or changelog

## First Public Scope

This repository is currently a public-safe planning and documentation repo.

Private runtime, backend execution, upload processing, and model keys should stay on the private Pi5/server side.

## Daily Build Rule

```text
1. Check current system
2. Pick one small task
3. Backup
4. Build
5. Test
6. Save result
7. Write log
```
